# lm mysql Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add `lm mysql`, `lm mysql init`, and `lm mysql user` so local MySQL setup and test-user creation can be completed inside `lm-tool`.

**Architecture:** Route the new `mysql` command family from `src/cli.js`, keep command behavior in a dedicated `src/commands/mysql.js`, and reuse the existing executor for all `mysql` client subprocesses. Persist connection settings under `config.mysql`, resolve migrations from `projects.server`, and extend the prompt layer only for the new interactive inputs and selections.

**Tech Stack:** Node.js ESM, built-in `node:test`, MySQL CLI subprocesses, `bcryptjs`, JSON config storage.
