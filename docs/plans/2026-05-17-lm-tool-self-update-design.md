# lm-tool 自更新与命令输出设计

## 目标

为 `lm-tool` 增加执行前自更新检查，并统一所有外部命令的中文提示与结束输出格式。

## 范围

本次改动覆盖以下行为：

- `lm`
- `lm help`
- `lm init`
- `lm build`
- `lm build server`
- `lm build web`
- `lm build admin`

本次改动不改变业务仓库的构建命令本身，只增加执行前的 `lm-tool` 自更新检查和输出规范。

## 自更新规则

### 触发范围

每次执行以下命令前，都要先检查 `lm-tool` 自身仓库是否有更新：

- `lm`
- `lm help`
- `lm init`
- `lm build`
- `lm build server`
- `lm build web`
- `lm build admin`

### 检查目录

以当前 CLI 可执行入口所在目录作为 `lm-tool` 根目录；该目录同时也是 `lm.config.json` 的存放目录。

### 检查步骤

1. 判断 `lm-tool` 目录是否为 git 仓库，不是则跳过自更新检查。
2. 执行 `git fetch` 获取远端最新信息。
3. 比较当前分支与其上游分支是否存在“远端领先”。

### 更新行为

- 如果没有更新：直接继续原命令。
- 如果有更新且工作区干净：
  - 输出 `正在拉取 lm-tool 最新代码`
  - 执行 `git pull`
  - 成功后继续原命令
- 如果有更新但工作区有本地变更：
  - 提示 `lm-tool 有更新，但是 lm-tool 本地仓库有变更`
  - 用上下键让用户选择：
    - `回退本地变更并更新`
    - `跳过更新，继续执行当前命令`
  - 选择第一项时执行 `git restore .` 和 `git pull`
  - 选择第二项时不更新，直接继续执行原命令

## 业务仓库更新规则

- `server` / `web` / `admin` 仓库代码更新只发生在 `build` 命令里。
- `lm-tool` 自更新不触碰业务仓库。
- 构建前的中文提示要明确说明对象，例如：
  - `正在拉取 server 仓库最新代码`
  - `正在拉取 web 仓库最新代码`
  - `正在拉取 admin 仓库最新代码`

## init 克隆规则

`lm init` 在需要克隆仓库时，输出中文提示，例如：

- `正在克隆 server 仓库到 <路径>`
- `正在克隆 web 仓库到 <路径>`
- `正在克隆 admin 仓库到 <路径>`

如果 `git clone` 失败，则输出：

`仓库拉取失败，如无权限请联系 @幻仔`

## 输出规则

### 原始输出

所有外部命令都必须实时透传原始 stdout 和 stderr，不允许吞掉命令输出。

### 命令结束格式

每个外部命令执行结束后，都要追加统一尾部：

- 成功：`[INFO] <命令说明> 执行成功`
- 失败：`[INFO] <命令说明> 执行失败`
- 然后输出：`=======================`

适用范围包括：

- 自更新相关的 `git fetch`、`git restore`、`git pull`
- `init` 中的 `git clone`
- `build` 中的 `git pull`、`mvn clean package -DskipTests`、`npm install`、`npm run build`
- server 重启中的外部命令

内部非命令步骤如“查找 jar”“复制 jar”“创建目录”不强制追加该格式。

## 失败处理

- 自更新检查中的 `git fetch` 失败：
  - 保留原始错误输出
  - 追加 `[INFO] lm-tool 更新检查失败`
  - 追加 `=======================`
  - 停止当前命令，不继续执行
- `git restore` 或 `git pull` 失败：
  - 保留原始错误输出
  - 追加统一 `[INFO]` 和分隔线
  - 停止当前命令
- 业务构建任一步失败：
  - 立即停止当前流程
  - 保留原始输出
  - 追加统一 `[INFO]` 和分隔线

## 文案与文档

- 所有面向用户的提示统一使用中文。
- 文案要具体说明动作和对象，不使用模糊表述。
- 以下三处文档必须同步维护：
  - `lm help`
  - `README.md`
  - `AGENTS.md`

## AGENTS 约束

根目录 `AGENTS.md` 用于记录长期规则，不记录一次性实现细节。至少包含：

- 执行前先检查 `lm-tool` 自更新
- build 才负责更新业务仓库代码
- 所有外部命令实时输出原始信息
- 所有提示使用详细中文
- 所有外部命令结束后输出 `[INFO]` 和 `=======================`
